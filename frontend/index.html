How it works

1
Real travel data
I use the Google Maps Distance Matrix API to fetch actual driving times and distances between every pair of stops, not straight-line estimates.
2
TSP optimization
For up to 9 stops, I run Held-Karp dynamic programming, an exact algorithm that guarantees the mathematically optimal route. For 10 to 15 stops, I use nearest-neighbour seeding followed by 2-opt local search, finding a near-optimal solution in milliseconds. Both are tuned for fast response times.
3
Constraint-aware
Fixed start and end points are enforced throughout both algorithms. Held-Karp seeds only valid starting states, and 2-opt never crosses the pinned boundaries.
